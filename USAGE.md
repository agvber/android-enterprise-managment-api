# AMM Console 사용 가이드

## 1. 사전 준비

### 필요한 것
- **macOS** (Python 3 기본 포함) 또는 Python 3이 설치된 환경
- **Google 계정** (Android Management API 접근 권한 필요)
- **GCP 프로젝트** (Android Management API가 활성화된 프로젝트)

> Node.js는 코드를 수정/재빌드할 때만 필요합니다. 단순 사용 시에는 불필요합니다.

### GCP 프로젝트에서 API 활성화
1. [Google Cloud Console](https://console.cloud.google.com) 접속
2. 프로젝트 선택 또는 새 프로젝트 생성
3. **API 및 서비스 > 라이브러리**에서 아래 API 활성화:
   - `Android Management API`
   - `Cloud Resource Manager API` (프로젝트 목록 조회용, 선택사항)

## 2. 시작 방법

### 방법 A: 더블클릭으로 시작 (권장, npm 불필요)

1. `amm` 폴더 안의 **`start.command`** 파일을 **더블클릭**합니다.
2. 터미널이 열리고 브라우저가 자동으로 `http://localhost:8080`을 엽니다.
3. 끝! 바로 사용할 수 있습니다.

> macOS에서 "확인되지 않은 개발자" 경고가 뜨면:
> 파일을 우클릭 → **열기** 선택 → **열기** 클릭

### 방법 B: 터미널에서 직접 실행 (npm 불필요)

```bash
cd amm
python3 server.py
```

브라우저에서 `http://localhost:8080` 접속. 포트를 변경하려면:

```bash
python3 server.py 9090
```

### 방법 C: 개발자용 (코드 수정 시)

```bash
# 의존성 설치 (최초 1회)
npm install

# 개발 서버 (코드 수정 시 자동 반영)
npm run dev

# 또는 정적 빌드 후 실행
npm run build
python3 server.py
```

### 서버 종료

- `start.command`로 시작한 경우: **터미널 창을 닫거나** `Ctrl + C`
- 터미널에서 시작한 경우: `Ctrl + C`

---

## 3. 로그인

1. 브라우저에서 접속하면 로그인 페이지가 표시됩니다.
2. **"Google 계정으로 인증"** 버튼 클릭 → 새 탭에서 Google 로그인 페이지 열림
3. Google 계정으로 로그인 후 권한 동의
4. 콜백 페이지에 표시되는 **Authorization Code**를 복사
5. 원래 탭으로 돌아와서 입력란에 Code 붙여넣기
6. **"로그인 완료"** 클릭

> 토큰은 브라우저 localStorage에 저장되며, 만료 시 자동 갱신됩니다.
> 로그아웃은 대시보드 좌측 하단의 **"로그아웃"** 버튼을 클릭합니다.

---

## 4. Enterprise 설정

로그인 후 가장 먼저 Enterprise를 연결해야 합니다.

### 기존 Enterprise 연결
1. **GCP 프로젝트 선택**: 프로젝트 목록이 자동으로 표시됩니다. 클릭하여 선택하거나 직접 입력합니다.
2. **"Enterprise 조회"** 클릭 → 해당 프로젝트의 Enterprise 목록이 표시됩니다.
3. 사용할 Enterprise를 클릭하면 선택됩니다.

### 새 Enterprise 생성
1. GCP 프로젝트 ID를 입력합니다.
2. **"Enterprise Signup URL 생성"** 클릭 → 새 탭에서 등록 페이지 열림
3. 등록 완료 후 URL에서 `signupUrlName`과 `enterpriseToken` 파라미터를 복사
4. 각각 입력란에 붙여넣고 **"Enterprise 등록 완료"** 클릭

---

## 5. 정책 관리

Enterprise를 선택한 후, 기기에 적용할 정책을 만들어야 합니다.

### 정책 만들기
1. 좌측 메뉴에서 **"정책 관리"** 클릭
2. **Policy ID** 입력 (예: `default`, `kiosk`, `field-device` 등)
3. **UI 편집** 모드에서 원하는 설정을 토글/선택:
   - **앱 관리**: 설치할 앱 추가, 설치 유형/권한/업데이트 설정
   - **보안**: 화면 캡처, 카메라, 디버깅, 암호화 등
   - **비밀번호**: 잠금 비밀번호 요구사항
   - **기기 기능**: Bluetooth, USB, Wi-Fi, SMS 등 제한
   - **시스템 업데이트**: 자동/시간대 지정/연기
   - **상태 보고**: 하드웨어/소프트웨어/메모리/전원 이벤트 보고 활성화
   - **잠금화면**: 카메라, 알림, 생체인식 비활성화
4. 또는 **JSON** 모드로 전환하여 직접 JSON 편집
5. **"정책 저장 / 업데이트"** 클릭 (확인 다이얼로그 표시)

### 정책 수정
1. 하단 **등록된 정책** 목록에서 **"편집"** 클릭 (현재 편집 중인 내용 대체 경고)
2. 편집기에 불러온 후 수정
3. **"정책 저장 / 업데이트"** 클릭

### 정책 삭제
- 정책 목록에서 **"삭제"** 클릭 (되돌릴 수 없음 경고)

> **팁**: 상태 보고 설정을 활성화해야 기기 상세에서 CPU, 온도, 전원 이벤트 등을 확인할 수 있습니다.

---

## 6. 기기 등록

### QR 코드로 기기 등록
1. 좌측 메뉴에서 **"기기 등록"** 클릭
2. 적용할 정책을 목록에서 클릭하여 선택 (또는 직접 입력)
3. **"등록 토큰 생성"** 클릭 → QR 코드와 토큰 값이 표시됩니다
4. Android 기기에서:
   - 기기를 **Factory Reset** (초기화) 합니다
   - "Welcome" 화면에서 **화면을 6번 탭**합니다
   - QR 코드 리더가 열리면 **QR 코드를 스캔**합니다
   - 기기가 자동으로 프로비저닝됩니다

> 토큰은 **24시간** 후 만료됩니다. 만료 시 새로 생성하세요.

---

## 7. 기기 관리

### 기기 목록 확인
1. 좌측 메뉴에서 **"기기 관리"** 클릭 → 등록된 기기 목록 표시
2. **검색**: 상단 검색창에서 IMEI, S/N, 모델명 등으로 검색
3. **필터**: 상태별(ACTIVE, PROVISIONING 등), 정책별 드롭다운 필터

### 기기 상세 보기
- 기기 카드에서 **"📋 상세"** 클릭 → 우측 슬라이드 패널:
  - 업타임, 하드웨어/소프트웨어 정보, 네트워크 정보
  - CPU 사용률 그래프 (코어별 라인 + 평균)
  - 코어별 CPU 바 차트 (현재 시점)
  - 온도 추이 그래프 (배터리/CPU/GPU/스킨)
  - 전원 이벤트 히스토리
  - 정책 미준수 항목
  - 원본 JSON 데이터 (펼치기)
- 패널 외부 클릭 또는 X 버튼으로 닫기

### 원격 명령
각 기기 카드의 버튼으로 명령을 전송합니다. 모든 명령은 확인 다이얼로그가 표시됩니다.

| 버튼 | 동작 |
|------|------|
| 🔒 **잠금** | 기기를 즉시 잠금 |
| 🔄 **재부팅** | 기기 재부팅 |
| 🔑 **비밀번호** | 새 비밀번호 입력 → 비밀번호 재설정 |
| 🗑️ **삭제** | 기기를 Enterprise에서 제거 (되돌릴 수 없음) |

## 8. 로그아웃

- 대시보드 좌측 사이드바 하단의 **"로그아웃"** 클릭
- localStorage의 토큰이 삭제되고 로그인 페이지로 이동합니다

## 자주 묻는 질문

### Q: `file://`로 열면 안 되나요?
A: 안 됩니다. Google OAuth 토큰 교환 API가 CORS로 `file://` 프로토콜을 차단합니다. `npx serve out` 등 HTTP 서버로 서빙해야 합니다.

### Q: 기기 상세에서 CPU/온도 그래프가 안 보여요
A: 정책의 **상태 보고** 설정에서 "하드웨어 상태 보고"를 활성화해야 합니다. 활성화 후 기기가 다음 보고 주기에 데이터를 전송하면 표시됩니다.

### Q: 토큰이 만료되면 어떻게 하나요?
A: Refresh Token이 있으면 자동 갱신됩니다. 자동 갱신이 실패하면 다시 로그인하세요.

### Q: GPS 위치를 확인할 수 있나요?
A: Android Management API에는 GPS 위치 조회 기능이 없습니다. 위치 보고용 앱을 별도로 만들어서 정책에 강제 설치(FORCE_INSTALLED)로 추가해야 합니다.

### Q: Enterprise 이름 형식이 뭔가요?
A: `enterprises/LC` + 영숫자 형태입니다. 예: `enterprises/LC01abc234`
